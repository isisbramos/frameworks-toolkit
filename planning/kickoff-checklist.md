# Kickoff Checklist

## When to use
When starting any new initiative, project, or quarter. Ensures alignment before work begins.

## Before the Kickoff

- [ ] Problem statement is written and validated
- [ ] - [ ] Success metrics are defined
- [ ] - [ ] Stakeholders are identified
- [ ] - [ ] Team roles are clear
- [ ] - [ ] Timeline expectations are set (even if rough)

- [ ] ## During the Kickoff

- [ ] - [ ] Share the "why" — context and motivation
- [ ] - [ ] Walk through the problem statement
- [ ] - [ ] Align on scope — what's in and what's out
- [ ] - [ ] Define success criteria
- [ ] - [ ] Agree on communication cadence
- [ ] - [ ] Identify risks and unknowns
- [ ] - [ ] Set next steps and owners

- [ ] ## After the Kickoff

- [ ] - [ ] Share meeting notes async
- [ ] - [ ] Create tracking artifacts (board, doc, etc.)
- [ ] - [ ] Schedule first check-in
- [ ] - [ ] Start discovery or execution based on phase

- [ ] ## Tips

- [ ] - Keep it under 60 minutes
- [ ] - The kickoff is about alignment, not decisions
- [ ] - If the problem isn't clear yet, the kickoff should focus on the discovery plan
