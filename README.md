# frameworks-toolkit

My open-source PM toolkit — templates, frameworks, and thinking tools I actually use.

Not theoretical. Everything here came from real problems I was trying to solve.

---

## 📦 What's inside

### Discovery

| File | Use when |
|------|----------|
| [opportunity-solution-tree.md](discovery/opportunity-solution-tree.md) | Mapping opportunities before jumping to solutions |
| [problem-statement-canvas.md](discovery/problem-statement-canvas.md) | Aligning the team on what problem we're actually solving |
| [jobs-to-be-done.md](discovery/jobs-to-be-done.md) | Understanding user motivation beyond surface-level requests |

### Planning

| File | Use when |
|------|----------|
| [kickoff-checklist.md](planning/kickoff-checklist.md) | Starting any new initiative or quarter |
| [now-next-later.md](planning/now-next-later.md) | Roadmap communication without fake dates |
| [retrospective.md](planning/retrospective.md) | End of sprint/cycle reflection |

### Communication

| File | Use when |
|------|----------|
| [prd-template.md](communication/prd-template.md) | Writing product specs that people actually read |
| [stakeholder-update-template.md](communication/stakeholder-update-template.md) | Weekly/bi-weekly async updates |

---

## 🧠 Philosophy

I use frameworks as thinking scaffolding, not bureaucracy.

If a framework isn't helping me think more clearly, I don't use it.
