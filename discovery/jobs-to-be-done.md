# Jobs to Be Done (JTBD)

## When to use
When you need to understand user motivation beyond surface-level feature requests. JTBD helps you uncover *why* users hire your product.

## The Job Statement Format

> When **[situation]**, I want to **[motivation]**, so I can **[expected outcome]**.
>
> ## Template
>
> ### Functional Job
> > What is the user trying to accomplish?
> >
> > ### Emotional Job
> > > How does the user want to feel?
> > >
> > > ### Social Job
> > > > How does the user want to be perceived?
> > > >
> > > > ### Related Jobs
> > > > > What other jobs is the user trying to get done in the same context?
> > > > >
> > > > > ## Interview Questions
> > > > >
> > > > > - What were you trying to do when you first looked for a solution?
> > > > > - - What was the situation that triggered your search?
> > > > >   - - What alternatives did you consider?
> > > > >     - - What made you choose this over the alternatives?
> > > > >       - - What would you go back to if this didn't exist?
> > > > >        
> > > > >         - ## Tips
> > > > >        
> > > > >         - - Focus on the job, not the product
> > > > >           - - Jobs are stable over time — solutions change
> > > > >             - - Look for struggling moments as signals of unmet jobs
> > > > >               - - Pair with opportunity mapping for maximum clarity
